Git & Github

About Github

Github is a company offering a cloud-based git repository that helps developers store, manage, track and control changes to their code.

Not let's understand two terms related to Github:

  • Version Control: Version control also refered as source control, is the process/method of tracking and managing changes/versions to the source code.
  • Git: Git is a tool used for version control. In simple words, git is used for managing source codes in a cloud repository.

Commands used in Git:

  • git init: Used to initialize current directory as a git repository
  • git clone: Clone a git repository to local machine
  • git add: Add files to staging area
  • git commit: Commit the staged changes
  • git push: Push the changes to the repository
  • git pull: Fetch the files from the specified repository
  • git status: List files staged, unstaged and untracked
NB: The commands listed above are the ones used frequently but not the only commands.

HTML Syntax

Simple HTML Document 

        
          <!DOCTYPE html> 
          <html>
          <head>
            <title>Sample Page Title</title>
          </head>
          <body>
            Sample Page Body
          </body>
          </html>

Explanation

DOCTYPE is the document type, used to tell the browser to process html files.

<HTML>..</html> is the opening and ending of html document.

<head>..</head> is the head of a document.

<title>..</title> contains the title of any document. Title tag is always placed inside head tag.

<body>..</body> contains the body of a document. Body refers to content of the page. Anything that are displayed in a webpage is placed within body opening and closing tag.

HTML Elements

Anything from start tag to end tag and the content between those tags are known as elements. Example: <html><body><p>

Nested Elements

An element within another element is known as nested element. Examples: <head><body>

Empty Elements

Elements with no closing tags are known as empty elements. Examples: <br><hr>
