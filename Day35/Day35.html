PHP Exceptions

An exception is an object that describes an error or unexpected behavior of a PHP script.

Throwing an Exception

We can use throw statement allows a user-defined function or method to throw an exception.

After throw statement is executed, no other statements below it will be executed.

Also, after an exception is thrown, the exception must be caught using catch block from try catch statement.

The try...catch Statement

The try...catch statement is used to catch exceptions and continue the process.

Syntax:

        try {
          // code that can throw exceptions
        } catch(Exception $e) {
          // code that runs when an exception is caught
        }

The try...catch...finally Statement

The try...catch...finally statement is used to catch exceptions.

The code in finally block runs regardless of whether an exception was caught or not.

The catch block is optional if finally is declared.

Syntax:

        try {
          // code that can throw exceptions
        } catch(Exception $e) {
          // code that runs when an exception is caught
        }

The Exception Object

The Exception Object contains information about any error that the function encountered.

Syntax:

        new Exception(messsage, code, previous)

Methods used to get information about exception:

  • getMessage()
  • getPrevious()
  • getCode()
  • getFile()
  • getLine()

PHP OOP

OOP stands for Object Oriented Programming. Introduced from PHP5, OOP is faster and easier to execute.

Why OOP?

  • Faster and easier to execute
  • Provides a clear structure for the programs
  • Makes the code easier to maintain, modify and debug

Terms Related to OOP

  • Class: It is the template for objects
  • Object: An instance of a class
  • Member Variables: Variables defined inside a class known as properties.
  • Member Functions:Function defined inside a class known as methods.
  • Inheritance: An OOP concept that allows to inherit functions and variables in a child class from a parent class.
  • Parent Class:A class that is inherited by another class.
  • Child Class:A class that inherits another class.
  • Polymorphism:An OOP concept that makes a function reusable for different purposes.
  • Overriding:A type of polymorphism where the definition of functions inherited from parent classes can be modified.
  • Data Abstraction:Any representation of data where the implementation details are hidden.
  • Encapsulation:An OOP concept where all the data and functions are encapsulated together to form an object.
  • Constructor:A special type of function which is called automatically when an instance is created.
  • Destructor:A special type of function which is called automatically when an object is deleted or goes out of scope.

PHP Classes and Objects

Classes

A class is a template for objects, and an object is an instance of class.

A class can be defined using the class keyword, followed by the name of class and a pair of curly braces ({}).

Syntax:

        class ClassName {
          // code goes here...
        }

Objects

Objects are created using new keyword followed by the name of the class.

Syntax:

        new ClassName();

The $this Keyword

The $this keyword refers to the current object, and is only available inside methods.

PHP Constructor

Constructor functions are special type of functions which are called automatically whenever an object is created.

A special function called __construct() is used to define a constructor in PHP.

Syntax:

        function __construct($par1, $par2) {
          // code goes here
        }

PHP Destructor

We can define a destructor function using __destruct().

The destruct function is called automatically at the end of the script.

Syntax:

        function __destruct() {
          // code goes here...
        }

PHP Access Modifiers

Properties and methods can have access modifiers which control from where they can be accessed.

Three Access Modifiers: public, protected, private

PHP Inheritance

Inheritance in OOP means a class can be derived by another class.

PHP class definitions can be inherited using extends clause.

Syntax:

        class Child extends Parent {
          // definition body
        }

What happens after Inheritance?

  • Child class automatically will have access to public and protected member variables and methods
  • Child class can have their own variables and methods apart from the parent class.

Function Overriding

The inherited methods can be overridden in child classes using the same name but different definition.

The final Keyword

The final keyword can be used to prevent class inheritance or to prevent method overriding.

PHP Class Constants

Class contants can be used to define constant data within a class

The const keyword is used to declare a class constant.

Syntax:

        const ClassName {
          const MESSAGE = "This is a message";
        }

We can access the constant from outside the class using the classname followed by scope resolution operator (::) followed by the constant name.

Example: echo className::MESSAGE;

PHP Abstraction

Abstraction is an OOP concept used to create abstract classes and methods.

Abstract classes and methods mean the parent class will only have declaration but the child classes need to do the definition.

An abstract class is any class that contains at least one abstract method.

We can use abstract keyword to define any abstract class or method.

Syntax:

        abstract class ParentClass {
          abstract public method method1();
          abstract public function method2($number);
        }

PHP Interfaces

Interfaces are used to specify what methods a class should implement.

Interfaces are declared using the interface keyword.

Syntax:

        interface InterfaceName {
          public function method1();
          public function method2($number);
        }

Interfaces vs. Abstract Classes

Interface are similar to abstract classes. However, there are some differences:

  • Interfaces cannot have properties, while abstract classes can.
  • All interface methods must be public, while abstract class methods can be public or protected
  • All methods in an interface is abstract by default, so the abstract keyword is not necessary.
  • Classes can implement an interface while inheriting from another class at the same time

Using Interfaces

To implement an interface, a class must use the implements keyword.

Example:class NewClass implements InterfaceName {}

NB: Any class implementing an Interface must implement all of the interface's methods.

PHP Traits

Since, PHP only supports single inheritance; a child class can only inherit one single parent, there may be times when we want to use methods from multiple classes.

OOP traits solve this problem. OOP Traits are used to declare methods that can be used in multiple classes.

Traits can have methods and abstract methods that can be used in multiple classes, and the methods can have any access modifier (public, private or protected).

Traits are declared using trait keyword

Syntax:

        trait TraitName {
          // code goes here...
        }

Traits are used, with use keyword.
