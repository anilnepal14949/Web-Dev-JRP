BEM Naming Standards

BEM (Block-Element-Modifier) is a naming convention standard for naming CSS classes.

Why use BEM?

  • Purpose or function of class is clear
  • Well communicated component structure
  • Consistency in styling selectors

How BEM works?

  • Block: A block is the outermost element of the component (parent)
  • Element: Nested inside the component (one or more children)
  • Modifier: Any block that needs to be modified.
Syntax: [block]__[element]--[modifier]

Component without elements

<button class="btn"></button> We can select the button using below: .btn {}

Component with elements

<figure class="project"> <img class="project__img" src="img.jpg"> <figcaption class="project__caption"> This is a caption </figcaption> </figure> Styles declaration: .project {} .project__img {} .project__caption {}

Component with modifier

<figure class="project"> <img class="project__img project__img--rounded" src="img.jpg"> <figcaption class="project__caption project__caption--fonts-changed"> This is a caption </figcaption> </figure> Styles declaration: .project {} .project__img {} .project__img--rounded {} .project__caption {} .project__caption--fonts-changed {}
